<div id="what-is-extract">
  ## Was ist `extract()`?
</div>

```typescript
page.extract("extract the name of the repository");
```

`extract` extrahiert strukturierte Daten von einer Webseite. Sie können Ihr Schema mit [zod](https://github.com/colinhacks/zod) (TypeScript) oder [pydantic](https://github.com/pydantic/pydantic) (Python) definieren. Wenn Sie kein Schema definieren möchten, können Sie `extract` auch nur mit einer [Aufforderung in natürlicher Sprache](#extract-with-just-a-prompt) aufrufen oder `extract` [ohne Parameter](#extract-with-no-parameters) verwenden.

<div id="why-use-extract">
  ## Warum `extract()` verwenden?
</div>

<CardGroup cols={2}>
  <Card title="Structured" icon="brackets-curly" href="#list-of-objects-extraction">
    Wandeln Sie unübersichtliche Webseitendaten in saubere, schemakonforme Objekte um.
  </Card>

  <Card title="Resilient" icon="dumbbell" href="#extract-with-context">
    Entwickeln Sie robuste Extraktionen, die nicht ausfallen, wenn sich die Website ändert.
  </Card>
</CardGroup>

<Note>
  Für TypeScript werden die extract-Schemata mit Zod-Schemata definiert.

  Für Python werden die extract-Schemata mit Pydantic-Modellen definiert.
</Note>

<div id="using-extract">
  ## Verwendung von `extract()`
</div>

<div id="single-object-extraction">
  ### Extraktion eines einzelnen Objekts
</div>

So könnte ein `extract`-Aufruf für ein einzelnes Objekt aussehen:

<CodeGroup>
  ```typescript TypeScript
  const item = await page.extract({
    instruction: "extract the price of the item",
    schema: z.object({
      price: z.number(),
    }),
  });
  ```

  ```python Python
  class Extraction(BaseModel):
      price: float

  item = await page.extract(
      "extract the price of the item", 
      schema=Extraction
  )
  ```
</CodeGroup>

Das Ausgabeschema sieht folgendermaßen aus:

```Example
{ price: number }
```

<div id="list-of-objects-extraction">
  ### Extraktion einer Objektliste
</div>

So könnte ein Aufruf von `extract()` für eine Liste von Objekten aussehen.

<CodeGroup>
  ```typescript TypeScript
  const apartments = await page.extract({
    instruction:
      "Extract ALL the apartment listings and their details, including address, price, and square feet.",
    schema: z.object({
      list_of_apartments: z.array(
        z.object({
          address: z.string(),
          price: z.string(),
          square_feet: z.string(),
        }),
      ),
    })
  })

  console.log("the apartment list is: ", apartments);
  ```

  ```python Python
  class Apartment(BaseModel):
      address: str
      price: str
      square_feet: str

  class Apartments(BaseModel):
      list_of_apartments: list[Apartment]

  apartments = await page.extract(
      "Extract ALL the apartment listings and their details as a list, including address, price, and square feet for each apartment",
      schema=Apartments
  )

  print("the apartment list is: ", apartments)
  ```
</CodeGroup>

Dein Ausgabeschema sieht folgendermaßen aus:

```Example
list_of_apartments: [
    {
      address: "street address here",
      price: "$1234.00",
      square_feet: "700"
    },
    {
        address: "another address here",
        price: "1010.00",
        square_feet: "500"
    },
    ...
]
```

<div id="prompt-only-extraction">
  ### Extraktion nur mit Prompt
</div>

Sie können `extract` mit einem reinen natürlichsprachigen Prompt aufrufen:

<CodeGroup>
  ```typescript TypeScript
  const result = await page.extract("extract the name of the repository");
  ```

  ```python Python
  result = await page.extract("extract the name of the repository")
  ```
</CodeGroup>

Wenn Sie `extract` nur mit einem Prompt aufrufen, sieht das Ausgabeschema so aus:

```Example
{ extraction: string }
```

<div id="extract-with-no-parameters">
  ### Extract ohne Parameter
</div>

So können Sie `extract` ohne Parameter aufrufen.

<CodeGroup>
  ```typescript TypeScript
  const pageText = await page.extract();
  ```

  ```python Python
  page_text = await page.extract()
  ```
</CodeGroup>

Ausgabeschema:

```Example
{ page_text: string }
```

Ein Aufruf von `extract` ohne Parameter gibt eine hierarchische Baumdarstellung des Root‑DOM zurück. Dies wird nicht durch ein LLM verarbeitet. Es sieht etwa so aus:

<div id="best-practices">
  ## Bewährte Vorgehensweisen
</div>

<div id="extract-with-context">
  ### Mit Kontext extrahieren
</div>

Sie können Ihrem Schema zusätzlichen Kontext hinzufügen, damit das Modell die Daten präziser extrahiert.

<CodeGroup>
  ```typescript TypeScript
  const apartments = await page.extract({
    instruction:
      "Extract ALL the apartment listings and their details, including address, price, and square feet.",
    schema: z.object({
      list_of_apartments: z.array(
        z.object({
          address: z.string().describe("the address of the apartment"),
          price: z.string().describe("the price of the apartment"),
          square_feet: z.string().describe("the square footage of the apartment"),
        }),
      ),
    })
  })
  ```

  ```python Python
  class Apartment(BaseModel):
      address: str = Field(..., description="the address of the apartment")
      price: str = Field(..., description="the price of the apartment")
      square_feet: str = Field(..., description="the square footage of the apartment")

  class Apartments(BaseModel):
      list_of_apartments: list[Apartment]

  apartments = await page.extract(
      "Extract ALL the apartment listings and their details as a list. For each apartment, include: the address of the apartment, the price of the apartment, and the square footage of the apartment",
      schema=Apartments
  )
  ```
</CodeGroup>

<div id="link-extraction">
  ### Link-Extraktion
</div>

<Note>
  Um Links oder URLs zu extrahieren, musst du in der TypeScript-Version von Stagehand das entsprechende Feld als `z.string().url()` definieren.
  In Python definierst du es als `HttpUrl`.
</Note>

So könnte ein `extract`-Aufruf zum Extrahieren eines Links oder einer URL aussehen. Das funktioniert auch für Bildlinks.

<CodeGroup>
  ```typescript TypeScript
  const extraction = await page.extract({
    instruction: "extract the link to the 'contact us' page",
    schema: z.object({
      link: z.string().url(), // note the usage of z.string().url() here
    }),
  });

  console.log("the link to the contact us page is: ", extraction.link);
  ```

  ```python Python
  class Extraction(BaseModel):
      link: HttpUrl # note the usage of HttpUrl here

  extraction = await page.extract(
      "extract the link to the 'contact us' page", 
      schema=Extraction
  )

  print("the link to the contact us page is: ", extraction.link)
  ```
</CodeGroup>

<Tip>
  In Stagehand werden Links extrahiert, indem das LLM eine ID auswählt. Stagehand schlägt diese ID in einer Zuordnung von IDs -> URLs nach. Beim Protokollieren des LLM-Traces solltest du IDs sehen. Die tatsächlichen URLs sind im finalen `ExtractResult` enthalten.
</Tip>

<div id="troubleshooting">
  ## Problembehebung
</div>

<AccordionGroup>
  <Accordion title="Empty or partial results">
    **Problem**: `extract()` gibt leere oder unvollständige Daten zurück

    **Lösungen**:

    - **Prüfen Sie die Eindeutigkeit Ihrer Anweisung**: Stellen Sie sicher, dass Ihre Anweisung präzise ist und genau beschreibt, welche Daten extrahiert werden sollen
    - **Verifizieren Sie das Vorhandensein der Daten**: Verwenden Sie zunächst `page.observe()`, um zu bestätigen, dass die Daten auf der Seite vorhanden sind
    - **Auf dynamische Inhalte warten**: Wenn die Seite Inhalte dynamisch lädt, verwenden Sie vor dem Extrahieren `page.act("wait for the content to load")`

    **Lösung: Vor dem Extrahieren auf Inhalte warten**

    <CodeGroup>
      ```typescript TypeScript
      // Wait for content before extracting
      await page.act("wait for the product listings to load");
      const products = await page.extract({
        instruction: "extract all product names and prices",
        schema: z.object({
          products: z.array(z.object({
            name: z.string(),
            price: z.string()
          }))
        })
      });
      ```

      ```python Python
      # Wait for content before extracting
      await page.act("wait for the product listings to load")
      products = await page.extract(
          "extract all product names and prices",
          schema=ProductList
      )
      ```
    </CodeGroup>
  </Accordion>

  <Accordion title="Schema validation errors">
    **Problem**: Schema-Validierungsfehler oder Typinkonsistenzen

    **Lösungen**:

    - **Optionale Felder verwenden**: Felder mit `z.optional()` (TypeScript) oder `Optional[type]` (Python) optional machen, wenn die Daten nicht immer vorhanden sind
    - **Flexible Typen verwenden**: Erwägen Sie `z.string()` statt `z.number()` für Preise, die möglicherweise Währungssymbole enthalten
    - **Beschreibungen hinzufügen**: Verwenden Sie `.describe()` (TypeScript) oder `Field(description="...")` (Python), um die Feldanforderungen für das Modell klarzustellen

    **Lösung: Flexibleres Schema**

    <CodeGroup>
      ```typescript TypeScript
      const schema = z.object({
        price: z.string().describe("price including currency symbol, e.g., '$19.99'"),
        availability: z.string().optional().describe("stock status if available"),
        rating: z.number().optional()
      });
      ```

      ```python Python
      class FlexibleProduct(BaseModel):
          price: str = Field(description="price including currency symbol, e.g., '$19.99'")
          availability: Optional[str] = Field(default=None, description="stock status if available")
          rating: Optional[float] = None
      ```
    </CodeGroup>
  </Accordion>

  <Accordion title="Inconsistent results">
    **Problem**: Extraktionsergebnisse variieren zwischen Durchläufen

    **Lösungen**:

    - **Anweisungen präziser formulieren**: Statt „extract prices“ besser „extract the numerical price value for each item“ verwenden
    - **Kontext in Schema-Beschreibungen nutzen**: Feldbeschreibungen hinzufügen, um das Modell zu führen
    - **Mit observe kombinieren**: Verwenden Sie `page.observe()`, um zuerst die Seitenstruktur zu verstehen

    **Lösung: Zuerst mit observe validieren**

    <CodeGroup>
      ```typescript TypeScript
      // Zuerst observe, um die Seitenstruktur zu verstehen
      const elements = await page.observe("find all product listings");
      console.log("Found elements:", elements.map(e => e.description));

      // Dann gezielt extrahieren
      const products = await page.extract({
        instruction: "extract name and price from each product listing shown on the page",
        schema: z.object({
          products: z.array(z.object({
            name: z.string().describe("the product title or name"),
            price: z.string().describe("the price as displayed, including currency")
          }))
        })
      });
      ```

      ```python Python
      # Zuerst observe, um die Seitenstruktur zu verstehen
      elements = await page.observe("find all product listings")
      print("Found elements:", [e.description for e in elements])

      # Dann gezielt extrahieren
      products = await page.extract(
          "extract name and price from each product listing shown on the page",
          schema=ProductSchema
      )
      ```
    </CodeGroup>
  </Accordion>

  <Accordion title="Performance issues">
    **Problem**: Extraktion ist langsam oder läuft in ein Timeout

    **Lösungen**:

    - **Umfang reduzieren**: Extrahieren Sie kleinere Datenblöcke in mehreren Aufrufen statt alles auf einmal
    - **Gezielte Anweisungen verwenden**: Geben Sie genau an, auf welchen Teil der Seite sich die Extraktion beziehen soll
    - **Paginierung berücksichtigen**: Bei großen Datensätzen jeweils nur eine Seite extrahieren
    - **Timeout erhöhen**: Verwenden Sie den Parameter `timeoutMs` für komplexe Extraktionen

    **Lösung: Große Extraktionen aufteilen**

    <CodeGroup>
      ```typescript TypeScript
      // Instead of extracting everything at once
      const allData = [];
      const pageNumbers = [1, 2, 3, 4, 5];

      for (const pageNum of pageNumbers) {
        await page.act(`navigate to page ${pageNum}`);
        
        const pageData = await page.extract({
          instruction: "extract product data from the current page only",
          schema: ProductPageSchema,
          timeoutMs: 60000 // 60 second timeout
        });
        
        allData.push(...pageData.products);
      }
      ```

      ```python Python
      # Instead of extracting everything at once
      all_data = []
      page_numbers = [1, 2, 3, 4, 5]

      for page_num in page_numbers:
          await page.act(f"navigate to page {page_num}")
          
          page_data = await page.extract(
              "extract product data from the current page only",
              schema=ProductPageSchema,
              timeout_ms=60000  # 60 second timeout
          )
          
          all_data.extend(page_data.products)
      ```
    </CodeGroup>
  </Accordion>
</AccordionGroup>
