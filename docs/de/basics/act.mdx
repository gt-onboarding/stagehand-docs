<div id="what-is-act">
  ## Was ist `act()`?
</div>

```typescript
page.act("click on add to cart")
```

`act` ermöglicht Stagehand, **einzelne** Aktionen auf einer Webseite auszuführen. Verwenden Sie es, um selbstheilende und deterministische Automatisierungen zu erstellen, die sich an Änderungen der Website anpassen.

<div id="why-use-act">
  ## Warum `act()` verwenden?
</div>

<div id="using-act">
  ## Verwendung von `act()`
</div>

Verwenden Sie `act`, um einzelne Aktionen in Ihrer Automatisierung auszuführen. So klicken Sie auf eine Schaltfläche:

<CodeGroup>
  ```typescript TypeScript
  await page.goto("https://example-store.com");
  await page.act("click the add to cart button");
  ```

  ```python Python
  await page.goto("https://example-store.com")
  await page.act("click the add to cart button")
  ```
</CodeGroup>

Mit `act` ist es am effektivsten, komplexe Aufgaben in kleine, einstufige Aktionen zu zerlegen. Wenn Sie mehrstufige Abläufe orchestrieren müssen, verwenden Sie mehrere `act`-Befehle oder `agent`.

<Accordion title="Empfohlene Aktionen">
  | Action | Example instruction |
  |--------|---------------------|
  | Click | `click the button` |
  | Fill | `fill the field with <value>` |
  | Type | `type <text> into the search box` |
  | Press | `press <key> in the search field` |
  | Scroll | `scroll to <position>` |
  | Select from dropdown | `select <value> from the dropdown` |
</Accordion>

<div id="advanced-configuration">
  ### Erweiterte Konfiguration
</div>

Für fortgeschrittene Anwendungsfälle können Sie zusätzliche Optionen konfigurieren:

<CodeGroup>
  ```typescript TypeScript
  // Dynamische Lebensmittelsuche mit erweiterten Optionen
  const foodItem = "organic quinoa";

  await page.act({
    action: "Type %foodItem% in the search box and press enter",
    variables: {
      foodItem: foodItem
    },
    modelName: "google/gemini-2.5-pro",
    modelClientOptions: {
      modelApiKey: process.env.GOOGLE_API_KEY,
    },
    iframes: true, // Bei Bedarf in iframes suchen
    domSettleTimeoutMs: 45000, // Länger auf dynamische Inhalte warten
    timeoutMs: 60000 // Verlängertes Timeout für langsam ladende Formulare
  });
  ```

  ```python Python
  # Dynamische Lebensmittelsuche mit erweiterten Optionen
  food_item = "organic quinoa"

  await page.act({
    "action": "Type %foodItem% in the search box and press enter",
    "variables": {
      "foodItem": food_item
    },
    "modelName": "google/gemini-2.5-pro",
    "modelClientOptions": {
      "modelApiKey": os.environ.get("GOOGLE_API_KEY")
    },
    "iframes": True, # Bei Bedarf in iframes suchen
    "domSettleTimeoutMs": 45000, # Länger auf dynamische Inhalte warten
    "timeoutMs": 60000 # Verlängertes Timeout für langsam ladende Formulare
  })
  ```
</CodeGroup>

<div id="ensure-reliable-actions">
  ### Zuverlässige Aktionen sicherstellen
</div>

Verwenden Sie `observe()`, um mögliche Aktionen auf der aktuellen Seite zu ermitteln und verlässlich zu planen. Die Funktion gibt eine Liste vorgeschlagener Aktionen zurück (mit Selektor, Beschreibung, Methode und Argumenten). Sie können eine erkannte Aktion direkt an `act` übergeben, um sie auszuführen.

<CodeGroup>
  ```typescript TypeScript
  const [action] = await page.observe("click the login button");

  if (action) {
    await page.act(action);
  }
  ```

  ```python Python
  results = await page.observe("click the login button")

  if results:
      await page.act(results[0])
  ```
</CodeGroup>

<Card title="Seiten mit observe() analysieren" icon="magnifying-glass" iconType="sharp-solid" href="/de/basics/observe">
  Planen Sie Aktionen mit `observe()`, bevor Sie sie mit `act` ausführen.
</Card>

<div id="reduce-model-costs">
  ### Modellkosten senken
</div>

Beobachtete Aktionen zwischenspeichern, um wiederholte LLM‑Aufrufe zu vermeiden und eine konsistente Ausführung sicherzustellen.

<CodeGroup>
  ```typescript TypeScript
  // Kostenoptimierte Aktionen mit Caching
  const actionCache = new Map();

  const getCachedAction = async (instruction: string) => {
    if (actionCache.has(instruction)) {
      return actionCache.get(instruction);
    }
    
    const [action] = await page.observe(instruction);
    actionCache.set(instruction, action);
    return action;
  };

  // Zwischengespeicherte Aktionen wiederverwenden
  const loginAction = await getCachedAction("click the login button");
  await page.act(loginAction);
  ```

  ```python Python
  # Kostenoptimierte Aktionen mit Caching
  action_cache = {}

  async def get_cached_action(instruction: str):
      if instruction in action_cache:
          return action_cache[instruction]
      
      results = await page.observe(instruction)
      if results:
          action = results[0]
          action_cache[instruction] = action
          return action
      
      return None

  # Zwischengespeicherte Aktionen wiederverwenden
  login_action = await get_cached_action("click the login button")
  if login_action:
      await page.act(login_action)
  ```
</CodeGroup>

<div id="secure-your-automations">
  ### Schützen Sie Ihre Automatisierungen
</div>

Variablen werden nicht an LLM-Anbieter weitergegeben. Verwenden Sie sie für Passwörter, API-Schlüssel und andere vertrauliche Daten.

<Note>
  Laden Sie vertrauliche Daten über Umgebungsvariablen aus `.env`-Dateien. Hartkodieren Sie niemals API-Schlüssel, Passwörter oder andere Secrets direkt in Ihrem Code.
</Note>

<CodeGroup>
  ```typescript TypeScript
  await page.act({
    action: "enter %username% in the email field",
    variables: {
      username: "user@example.com"
    }
  });

  await page.act({
    action: "enter %password% in the password field",
    variables: {
      password: process.env.USER_PASSWORD
    }
  });
  ```

  ```python Python
  # If using Python, set `use_api: true` in your Stagehand configuration

  await page.act(
    "enter %username% in the email field",
    variables={
        "username": "user@example.com"
    }
  )

  await page.act(
    "enter %password% in the password field",
    variables={
        "password": os.environ.get("USER_PASSWORD")
    }
  )
  ```
</CodeGroup>

<Warning>
  Beim Umgang mit vertraulichen Daten setzen Sie `verbose: 0` in Ihrer Stagehand-Konfiguration, um zu verhindern, dass Secrets in Logs erscheinen. Siehe den [Konfigurationsleitfaden](/de/configuration/browser) für weitere Details.
</Warning>

<div id="troubleshooting">
  ## Problembehandlung
</div>

<AccordionGroup>
  <Accordion title="Method not supported">
    **Problem**: `act` schlägt mit dem Fehler „method not supported“ fehl

    **Lösungen**:

    - Verwenden Sie klare, präzise Anweisungen für das gewünschte Ergebnis
    - Sehen Sie sich unsere [Evals](https://stagehand.dev/evals) an, um die besten Modelle für Ihren Anwendungsfall zu finden
    - Verwenden Sie [`observe()`](/de/basics/observe) und prüfen Sie, ob die resultierende Aktion in einer Liste erwarteter Aktionen enthalten ist

    **Lösung 1: Mit observe validieren**

    <CodeGroup>
      ```typescript TypeScript
      const prompt = "click the submit button";
      const expectedMethod = "click";

      try {
        await page.act(prompt);
      } catch (error) {
        if (error.message.includes("method not supported")) {
          // Observe the same prompt to get the planned action
          const [action] = await page.observe(prompt);
          
          if (action && action.method === expectedMethod) {
            await page.act(action);
          } else {
            throw new Error(`Unsupported method: expected "${expectedMethod}", got "${action?.method}"`);
          }
        } else {
          throw error;
        }
      }
      ```

      ```python Python
      prompt = "click the submit button"
      expected_method = "click"

      try:
          await page.act(prompt)
      except Exception as error:
          if "method not supported" in str(error):
              # Observe the same prompt to get the planned action
              results = await page.observe(prompt)
              
              if results and results[0].method == expected_method:
                  await page.act(results[0])
              else:
                  method = results[0].method if results else "unknown"
                  raise Exception(f'Unsupported method: expected "{expected_method}", got "{method}"')
          else:
              raise error
      ```
    </CodeGroup>

    **Lösung 2: Mit exponentiellem Backoff erneut versuchen**

    <CodeGroup>
      ```typescript TypeScript
      // Retry with exponential backoff for intermittent issues
      const prompt = "click the submit button";
      const maxRetries = 3;

      for (let attempt = 0; attempt <= maxRetries; attempt++) {
        try {
          await page.act(prompt, { timeoutMs: 10000 + (attempt * 5000) });
          break; // Success, exit retry loop
        } catch (error) {
          if (error.message.includes("method not supported") && attempt < maxRetries) {
            // Exponential backoff: wait 2^attempt seconds
            const delay = Math.pow(2, attempt) * 1000;
            console.log(`Retry ${attempt + 1}/${maxRetries} after ${delay}ms`);
            await new Promise(resolve => setTimeout(resolve, delay));
          } else {
            throw error;
          }
        }
      }
      ```

      ```python Python
      # Retry with exponential backoff for intermittent issues
      import asyncio

      prompt = "click the submit button"
      max_retries = 3

      for attempt in range(max_retries + 1):
          try:
              timeout = 10000 + (attempt * 5000)
              await page.act(prompt, {"timeoutMs": timeout})
              break  # Success, exit retry loop
          except Exception as error:
              if "method not supported" in str(error) and attempt < max_retries:
                  # Exponential backoff: wait 2^attempt seconds
                  delay = 2 ** attempt
                  print(f"Retry {attempt + 1}/{max_retries} after {delay}s")
                  await asyncio.sleep(delay)
              else:
                  raise error
      ```
    </CodeGroup>
  </Accordion>

  <Accordion title="Action failed or timed out">
    **Problem**: `act` läuft in ein Timeout oder kann die Aktion nicht abschließen (oft, weil das Element nicht gefunden wird)

    **Lösungen**:

    - Sicherstellen, dass die Seite vollständig geladen ist
    - Prüfen, ob Inhalte in iframes liegen: [Mehr über die Arbeit mit iframes](/de/best-practices/working-with-iframes)
    - Action-Timeout erhöhen
    - Zuerst `observe()` verwenden, um zu prüfen, ob das Element existiert

    <CodeGroup>
      ```typescript TypeScript
      // Timeout- und „Element nicht gefunden“-Probleme behandeln
      try {
        await page.act("click the submit button", { timeout: 30000 });
      } catch (error) {
        // Prüfen, ob die Seite vollständig geladen ist
        await page.waitForLoadState('domcontentloaded');
        
        // observe verwenden, um den Elementzustand zu prüfen
        const [element] = await page.observe("find the submit button");
        
        if (element) {
          console.log("Element gefunden, versuche spezifischere Anweisung");
          await page.act("click the submit button at the bottom of the form");
        } else {
          console.log("Element nicht gefunden, versuche alternativen Selektor");
          await page.act("click the button with text 'Submit'");
        }
      }
      ```

      ```python Python
      # Timeout- und „Element nicht gefunden“-Probleme behandeln
      try:
          await page.act("click the submit button", {"timeout": 30000})
      except Exception as error:
          # Prüfen, ob die Seite vollständig geladen ist
          await page.wait_for_load_state('domcontentloaded')
          
          # observe verwenden, um den Elementzustand zu prüfen
          results = await page.observe("find the submit button")
          
          if results:
              print("Element gefunden, versuche spezifischere Anweisung")
              await page.act("click the submit button at the bottom of the form")
          else:
              print("Element nicht gefunden, versuche alternativen Selektor")
              await page.act("click the button with text 'Submit'")
      ```
    </CodeGroup>
  </Accordion>

  <Accordion title="Incorrect element selected">
    **Problem**: `act` führt die Aktion auf dem falschen Element aus

    **Lösungen**:

    - Formulieren Sie die Anweisungen präziser: visuelle Merkmale, Position oder Kontext angeben
    - Verwenden Sie `observe()`, um vorab zu prüfen, welches Element ausgewählt wird
    - Kontextinformationen hinzufügen: „den Such-Button in der Kopfzeile“
    - Eindeutige Kennungen verwenden, wenn verfügbar

    <CodeGroup>
      ```typescript TypeScript
      // Präzisere Zielauswahl für Elemente
      // Statt:
      await page.act("click the button");

      // Konkreteren Kontext verwenden:
      await page.act("click the red 'Delete' button next to the user John Smith");

      // Oder zuerst mit observe eine Vorschau einholen:
      const [action] = await page.observe("click the submit button in the checkout form");
      if (action.description.includes("checkout")) {
        await page.act(action);
      }
      ```

      ```python Python
      # Präzisere Zielauswahl für Elemente
      # Statt:
      await page.act("click the button")

      # Konkreteren Kontext verwenden:
      await page.act("click the red 'Delete' button next to the user John Smith")

      # Oder zuerst mit observe eine Vorschau einholen:
      results = await page.observe("click the submit button in the checkout form")
      if results and "checkout" in results[0].description:
          await page.act(results[0])
      ```
    </CodeGroup>
  </Accordion>
