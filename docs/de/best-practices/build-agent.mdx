Stagehand stellt KI‑Agenten leistungsstarke Werkzeuge bereit, um einen Browser vollständig autonom zu steuern. Sieh dir unten an, wie ein Stagehand‑Agent eigenständig zu einer URL navigiert, Aktionen auf der Seite ausführt und strukturierte Daten extrahiert, um eine Frage zu beantworten.
Es gibt mehrere Möglichkeiten, mit Stagehand einen Agenten zu erstellen. Schauen wir uns ein paar davon an.

<div id="stagehand-mcp">
  ## Stagehand MCP
</div>

Das obige Beispiel zeigt einen Claude‑Agenten, der Stagehand zur Browsersteuerung nutzt. Zum Zeitpunkt der Erstellung dieses Textes wird [multimodales Tool‑Calling](https://sdk.vercel.ai/docs/ai-sdk-core/tools-and-tool-calling#multi-modal-tool-results) nur in Claude 3.5/3.7 Sonnet unterstützt.
Das bedeutet, dass Claude selbstständig erkennt, wann ein Browser‑Screenshot angefordert werden sollte, und diesen anschließend nutzen kann, um zu entscheiden, welche Aktionen als Nächstes auszuführen sind.

Besonders interessant ist, dass der Agent den Browserzustand beurteilen und Aktionen unabhängig voneinander ausführen kann!
Claude kann über den Browserzustand nachdenken, während Stagehand mit GPT‑4o‑mini oder einem Computer‑Use‑Modell Aktionen auf der Seite ausführen kann.
Stagehand ist sogar in der Lage zu erkennen, wann GPT‑4o‑mini und wann ein Computer‑Use‑Modell sinnvoll ist, z. B. bei der Erkennung von iframes.

<div id="stagehand-computer-use-models">
  ## Stagehand + Computer‑Use‑Modelle
</div>

Mit Stagehand nutzen Sie leistungsstarke Computer‑Use‑APIs von OpenAI und Anthropic mit nur einer einzigen Codezeile.

<CodeGroup>
  ```typescript TypeScript
  await page.goto("https://github.com/browserbase/stagehand");

  // Create a Computer Use agent with just one line of code!
  const agent = stagehand.agent({
  	provider: "openai",
  	model: "computer-use-preview"
  });

  // Use the agent to execute a task
  const result = await agent.execute("Extract the top contributor's username");
  console.log(result);
  ```

  ```python Python
  await page.goto("https://github.com/browserbase/stagehand-python")

  # Create a Computer Use agent with just one line of code!
  agent = stagehand.agent(
      model="computer-use-preview"
  )

  # Use the agent to execute a task
  result = await agent.execute("Extract the top contributor's username")
  print(result)
  ```
</CodeGroup>

<div id="sequential-tool-calling-open-operator">
  ## Sequenzielles Tool-Calling (Open Operator)
</div>

Im Januar 2025 hat Browserbase [Open Operator](https://operator.browserbase.com) veröffentlicht.
Open Operator kann den Browserzustand verstehen und entsprechend handeln, um größere Aufgaben wie „Bestell mir eine Pizza“ zu erledigen.
Es arbeitet, indem es Stagehand‑Tools nacheinander aufruft:

1. Wenn keine URL vorhanden ist, eine Standard‑URL aufrufen.
2. Den Browserzustand prüfen. Ein LLM entscheiden lassen, was als Nächstes zu tun ist.
3. Mit `page.act()` die vom LLM vorgeschlagene Aktion ausführen.
4. Wiederholen

<Excalidraw className="w-full" url="https://link.excalidraw.com/readonly/dKh5sB1gEM1EjVqRCGKn" />

Das Einbinden von `stagehand.agent` in deine Browser‑Automatisierung ist so einfach wie das Hinzufügen einer einzigen Codezeile:

<Note>
  Python unterstützt derzeit `stagehand.agent` mit Computer‑Use‑Agent (CUA)‑Modellen. Die Standardimplementierung folgt in Kürze.
</Note>

<CodeGroup>
  ```typescript TypeScript
  await stagehand.page.goto("https://github.com/browserbase/stagehand");

  // Open Operator verwendet das Standard-LLM aus der Stagehand-Konfiguration
  const operator = stagehand.agent();
  const { message, actions } = await operator.execute(
  	"Extrahiere den Benutzernamen des Top-Contributors"
  );

  console.log(message);
  ```
</CodeGroup>

<div id="replay-the-agents-actions">
  ### Aktionen des Agenten wiedergeben
</div>

Sie können die Aktionen des Agenten genauso wiedergeben, wie Sie es mit einem regulären Stagehand‑Agenten tun würden. Sie können die Aktionen sogar automatisch cachen, um unnötige LLM‑Aufrufe bei wiederholten Durchläufen zu vermeiden.

Verwenden wir die unten stehende Funktion `replay`, um die Aktionen in einer Stagehand‑Skriptdatei zu speichern. Diese reproduziert dieselben Schritte des Agenten und hat integriertes Caching der Aktionen.

<Accordion title="utils.ts">
  ```typescript
  import { AgentAction, AgentResult } from "@browserbasehq/stagehand";
  import { exec } from "child_process";
  import fs from "fs/promises";

  export async function replay(result: AgentResult) {
    const history = result.actions;
    const replay = history
      .map((action: AgentAction) => {
        switch (action.type) {
          case "act":
            if (!action.playwrightArguments) {
              throw new Error("No playwright arguments provided");
            }
            return `await page.act(${JSON.stringify(
              action.playwrightArguments
            )})`;
          case "extract":
            return `await page.extract("${action.parameters}")`;
          case "goto":
            return `await page.goto("${action.parameters}")`;
          case "wait":
            return `await page.waitForTimeout(${parseInt(
              action.parameters as string
            )})`;
          case "navback":
            return `await page.goBack()`;
          case "refresh":
            return `await page.reload()`;
          case "close":
            return `await stagehand.close()`;
          default:
            return `await stagehand.oops()`;
        }
      })
      .join("\n");

    console.log("Replay:");
    const boilerplate = `
  import { Page, BrowserContext, Stagehand } from "@browserbasehq/stagehand";

  export async function main(stagehand: Stagehand) {
      const page = stagehand.page
  	${replay}
  }
    `;
    await fs.writeFile("replay.ts", boilerplate);

    // Format the replay file with prettier
    await new Promise((resolve, reject) => {
      exec(
        "npx prettier --write replay.ts",
        (error: any, stdout: any, stderr: any) => {
          if (error) {
            console.error(\`Error formatting replay.ts: \${error}\`);
            reject(error);
            return;
          }
          resolve(stdout);
        }
      );
    });
  }
  ```
</Accordion>

Hier ist die Replay‑Ausgabe für eine Anweisung wie „Get me the stock price of NVDA“:

```typescript {14-22} replay.ts
import { Page, BrowserContext, Stagehand } from "@browserbasehq/stagehand";

export async function main({
  page,
  context,
  stagehand,
}: {
  page: Page; // Playwright Page with act, extract, and observe methods
  context: BrowserContext; // Playwright BrowserContext
  stagehand: Stagehand; // Stagehand instance
}) {
  await page.goto("https://www.google.com");

  // Replay nutzt standardmäßig zuerst Playwright, um unnötige LLM-Aufrufe zu vermeiden!
  // Wenn die Playwright-Aktion fehlschlägt, übernimmt Stagehand AI und heilt sich selbst
  await page.act({
    description: "The search combobox where users can type their queries.",
    method: "fill",
    arguments: ["NVDA stock price"],
    selector:
      "xpath=/html/body[1]/div[1]/div[3]/form[1]/div[1]/div[1]/div[1]/div[1]/div[2]/textarea[1]",
  });
  await page.extract(
    "the displayed NVDA stock price in the search suggestions",
  );
  await stagehand.close();
}
```
