Gute Prompts machen Stagehand zuverlässig. Schlechte Prompts führen zu Ausfällen. So erstellen Sie Prompts, die konsistent funktionieren.

<div id="act-method">
  ## act()-Methode
</div>

Verwende `act()` für einzelne Aktionen auf Webseiten. Jede Aktion sollte klar und präzise sein.

<CodeGroup>
```typescript TypeScript
// Gut – einzelne, spezifische Aktionen
await page.act("click the 'Add to Cart' button");
await page.act("type 'user@example.com' into the email field");

// Schlecht – mehrere Aktionen kombiniert
await page.act("fill out the form and submit it");
await page.act("login with credentials and navigate to dashboard");
```

```python Python

await page.act("click the 'Add to Cart' button")
await page.act("type 'user@example.com' into the email field")

# Schlecht – mehrere Aktionen kombiniert
await page.act("fill out the form and submit it")
await page.act("login with credentials and navigate to dashboard")
```
</CodeGroup>

<div id="choose-the-right-action-verbs">

</div>

- **Type** für Texteingabefelder
- **Select** für Dropdown-Menüs
- **Check/uncheck** für Kontrollkästchen

<div id="protect-sensitive-data">
  ### Vertrauliche Daten schützen
</div>

Variablen halten vertrauliche Informationen aus Prompts und Logs heraus.

<CodeGroup>
```typescript TypeScript
// Gut – sichere Vorgehensweise
await page.act({
  action: "enter %username% in the email field",
  variables: {
    username: "user@example.com"
  }
});

await page.act({
  action: "enter %password% in the password field", 
  variables: {
    password: process.env.USER_PASSWORD
  }
});

// Schlecht – unsichere Vorgehensweise
await page.act("type 'mySecretPassword123' into the password field");
```

```python Python
import os

# Gut – sichere Vorgehensweise
await page.act(
  "enter %username% in the email field",
  variables={
    "username": "user@example.com"
  }
)

await page.act(
  "enter %password% in the password field",
  variables={
    "password": os.environ.get("USER_PASSWORD")
  }
)

# Schlecht – unsichere Vorgehensweise
await page.act("type 'mySecretPassword123' into the password field")
```
</CodeGroup>

<div id="extract-method">
  ## Extract-Methode
</div>

Verwenden Sie `extract()`, um strukturierte Daten aus Seiten zu extrahieren. Definieren Sie klare Schemas und liefern Sie Kontext.

<div id="schema-best-practices">
  ### Best Practices für Schemas
</div>

Verwenden Sie aussagekräftige Feldnamen, passende Typen und präzise Beschreibungen. Feldbeschreibungen liefern Kontext, der dem Agent hilft, genau zu verstehen, was extrahiert werden soll.

<CodeGroup>
```typescript TypeScript
// Gut – aussagekräftige Namen, korrekte Typen und hilfreiche Beschreibungen
const productData = await page.extract({
  instruction: "Extract product information",
  schema: z.object({
    productTitle: z.string().describe("The main product name displayed on the page"),
    priceInDollars: z.number().describe("Current selling price as a number, without currency symbol"),
    isInStock: z.boolean().describe("Whether the product is available for purchase")
  })
});

// Schlecht – generische Namen, falsche Typen, keine Beschreibungen
const data = await page.extract({
  instruction: "Get product details", 
  schema: z.object({
    name: z.string(), // Zu generisch, kein Kontext
    price: z.string(), // Sollte eine Zahl sein
    stock: z.string() // Sollte ein Boolean sein, kein Kontext
  })
});
```

```python Python
from pydantic import BaseModel, Field

# Gut – aussagekräftige Namen, korrekte Typen und hilfreiche Beschreibungen
class ProductData(BaseModel):
    productTitle: str = Field(description="The main product name displayed on the page")
    priceInDollars: float = Field(description="Current selling price as a number, without currency symbol")
    isInStock: bool = Field(description="Whether the product is available for purchase")

productData = await page.extract(
  "Extract product information",
  schema=ProductData
)

# Schlecht – generische Namen, falsche Typen, keine Beschreibungen
class Data(BaseModel):
    name: str      # Zu generisch, kein Kontext
    price: str     # Sollte float sein, kein Kontext
    stock: str     # Sollte bool sein, kein Kontext

data = await page.extract(
  "Get product details",
  schema=Data
)
```
</CodeGroup>

<div id="handle-arrays-correctly">
  ### Arrays korrekt handhaben
</div>

Schemas immer in Objekte kapseln für zuverlässige Extraktion.

<CodeGroup>
```typescript TypeScript
// Gut – Array in Objekt gekapselt
const listings = await page.extract({
  instruction: "Extract all apartment listings",
  schema: z.object({
    apartments: z.array(z.object({
      address: z.string(),
      rent: z.number()
    }))
  })
});

// Schlecht – nacktes Array
const listings = await page.extract({
  instruction: "Extract apartment listings",
  schema: z.array(z.string()) // So nicht machen
});
```

```python Python
from pydantic import BaseModel
from typing import List

# Gut – Array in Objekt gekapselt
class Apartment(BaseModel):
    address: str
    rent: float

class Listings(BaseModel):
    apartments: List[Apartment]

listings = await page.extract(
  "Extract all apartment listings",
  schema=Listings
)

# Schlecht – nacktes Array (nicht unterstützt)
# So nicht machen – Arrays müssen in Objekten gekapselt sein
```
</CodeGroup>

<div id="use-proper-url-types">
  ### Verwenden Sie die richtigen URL-Typen
</div>

Geben Sie URL-Typen an, damit Stagehand URLs extrahiert. Ohne passende URL-Typen extrahiert Stagehand keine URLs.

<CodeGroup>
```typescript TypeScript
// Gut – weist Stagehand an, URLs zu extrahieren
const links = await page.extract({
  instruction: "Navigationslinks extrahieren",
  schema: z.object({
    links: z.array(z.object({
      text: z.string(),
      url: z.string().url() // Erforderlich für die URL-Extraktion
    }))
  })
});
```

```python Python
from pydantic import BaseModel, HttpUrl
from typing import List

# Gut – weist Stagehand an, URLs zu extrahieren
class Link(BaseModel):
    text: str
    url: HttpUrl  # Erforderlich für die URL-Extraktion

class Links(BaseModel):
    links: List[Link]

links = await page.extract(
  "Navigationslinks extrahieren",
  schema=Links
)
```
</CodeGroup>

<div id="observe-method">
  ## Observe-Methode
</div>

Verwende `observe()`, um umsetzbare Elemente zu identifizieren, bevor du auf ihnen agierst.

<div id="check-elements-first">
  ### Elemente zuerst prüfen
</div>

Stellen Sie sicher, dass die Elemente vorhanden sind, bevor Sie eine Aktion ausführen, um Fehler zu vermeiden.

<CodeGroup>
```typescript TypeScript
// Elemente zuerst prüfen
const loginButtons = await page.observe("Find the login button");

if (loginButtons.length > 0) {
  await page.act(loginButtons[0]);
} else {
  console.log("No login button found");
}
```

```python Python
# Elemente zuerst prüfen
login_buttons = await page.observe("Find the login button")

if len(login_buttons) > 0:
    await page.act(login_buttons[0])
else:
    print("No login button found")
```
</CodeGroup>

<div id="agent-method">
  ## Agent-Methode
</div>

Verwende `agent()` für komplexe, mehrstufige Workflows. Gib ausführliche Anweisungen und setze passende Grenzen.

<div id="navigate-first">
  ### Zuerst navigieren
</div>

Fügen Sie keine Navigation in Agent-Aufgaben ein. Behandeln Sie sie separat.

<CodeGroup>
```typescript TypeScript
// Gut – zuerst navigieren
await page.goto('https://amazon.com');
await agent.execute('Search for wireless headphones under $100 and add the best rated one to cart');

// Schlecht – Navigation in der Aufgabe
await agent.execute('Go to Amazon, search for headphones, and add one to cart');
```


</CodeGroup>

<div id="common-mistakes-to-avoid">
  ## Häufige Fehler, die Sie vermeiden sollten
</div>

- **Mehrere Aktionen kombinieren** – Beschränken Sie jeden `act()`-Aufruf auf eine einzelne Aktion
- **Vage Beschreibungen verwenden** – Geben Sie genau an, mit welchen Elementen interagiert werden soll  
- **Sensible Daten offenlegen** – Verwenden Sie für Zugangsdaten stets Variablen
- **Validierung überspringen** – Prüfen Sie die Ergebnisse, bevor Sie fortfahren

<div id="testing-your-prompts">
  ## Testen Ihrer Prompts
</div>

1. **Einfach beginnen** – Zuerst die grundlegende Funktionalität testen
2. **Komplexität schrittweise erhöhen** – Sich zu komplexen Workflows hocharbeiten
3. **Ergebnisse überwachen** – Logging nutzen, um zu verstehen, was passiert
4. **Aus Fehlern iterieren** – Prompts verfeinern, wenn sie nicht funktionieren
Denken Sie daran: Gutes Prompting ist iterativ. Im Zweifel lieber genauer als vager sein.
